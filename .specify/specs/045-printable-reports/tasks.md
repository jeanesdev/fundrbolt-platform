## Phase 3: User Story 1 — NPO Event Summary Report (Priority: P1) 🎯 MVP

**Goal**: NPO admin downloads a visually rich PDF from the event dashboard.

**Independent Test**: `GET /api/v1/admin/events/{event_id}/reports/event-summary` returns a valid PDF with charts.

### Backend for User Story 1

- [ ] T007 Create `backend/app/services/event_report_service.py`:
  - `EventReportService.__init__(db: AsyncSession)`
  - `async generate_pdf(event_id: UUID) -> bytes`
  - Calls `EventDashboardService.get_dashboard_summary` + `get_segment_breakdown` ×3 (table, guest, company)
  - Fetches `npo.branding.logo_url` via eager-loaded join (Event → NPO → NPOBranding); fetches logo with `report_utils.fetch_image_as_base64`
  - **R002**: All chart generation is implemented as private sync methods `_generate_revenue_chart(data) -> str`, `_generate_cashflow_chart(data) -> str`, `_generate_waterfall_chart(data) -> str`, `_generate_segment_chart(data, label) -> str` on the class
  - Each chart method: `matplotlib.use("Agg")` at module level; creates Figure, plots data with value annotations on bars/points, returns base64 PNG string using `BytesIO`
  - All charts include data labels for monochrome readability
  - **R003 (F003)**: Entire sync pipeline (all chart generation + WeasyPrint) is placed inside a single `_build_pdf(context_data)` sync function and run via `asyncio.get_running_loop().run_in_executor(None, _build_pdf_fn)` — this avoids blocking the event loop

  - Raises `RuntimeError` on generation failure (caught by endpoint)

  - Segment leaderboards: 3 separate sections (by table, by guest, by company) each with a horizontal bar chart + data table
  - Revenue generators section (if present)
  - Footer: "Generated by FundrBolt on {timestamp}"
  - All monetary values formatted as USD with thousand separators

## Phase 4: User Story 2 — Bid Card Labels (Priority: P2)

**Goal**: NPO Admin selects auction items, picks a label size, and downloads a Brady-compatible bid card PDF.

**Independent Test**: `POST /api/v1/admin/events/{event_id}/reports/bid-cards` with `label_size: "3x5"` returns a PDF with correct page dimensions.

### Backend for User Story 2

- [ ] T013 Create `backend/app/services/bid_card_service.py`:
  - `BidCardService.__init__(db: AsyncSession)`
  - `async generate_pdf(event_id: UUID, item_ids: list[UUID] | None, label_size: LabelSize) -> bytes`
  - Query: load `AuctionItem` records for event with `status = 'published'`; if `item_ids` provided, filter to those IDs; order by `bid_number` ascending; eagerly load `.media`
  - For each item: select first media where `media_type='image'` ordered by `display_order`, fetch image as base64 via `report_utils.fetch_image_as_base64`; if no image or fetch fails, fall back to plain text placeholder (no image tag)
  - **R002**: Inline module-level `def _generate_qr_base64(url: str) -> str` using `qrcode[pil]`, box_size=4, border=2, PNG to BytesIO → base64. No separate `qr_utils.py`.
  - Label size → CSS dimensions map: `"2x3"→"50.8mm 76.2mm"`, `"2x4"→"50.8mm 101.6mm"`, `"3x3"→"76.2mm 76.2mm"`, `"3x5"→"76.2mm 127mm"`
  - Renders `templates/reports/bid_cards.html` with all items + chosen dimensions
  - **R001**: Use `asyncio.get_running_loop().run_in_executor(None, _run)` for WeasyPrint
  - Raises `ValueError("no_items")` if no published items found after filtering
- [ ] T014 Create `backend/app/templates/reports/bid_cards.html`:
  - Jinja2 `{{ page_width }}` / `{{ page_height }}` injected into `@page { size: {{ page_width }} {{ page_height }}; margin: 3mm; }`
  - For each item: `.card` div occupying 100% of page, `page-break-after: always` (except last)
  - Card layout: lot number (bold, large, ~1/5 height), item title (truncated), auction type badge (Live/Silent), starting bid, est. value, image (scaled-to-fit, rounded corners), QR code (bottom-right corner), donated_by (if set)
  - Text: `overflow: hidden; text-overflow: ellipsis` everywhere
  - `<img>` tags use `src="data:image/png;base64,{{ item.image_b64 }}"` pattern
- [ ] T015 Wire T013 into `admin_reports.py`:
  - `POST /admin/events/{event_id}/reports/bid-cards`
  - `@require_role("super_admin", "npo_admin", "npo_staff")`
  - Request body: `BidCardRequest` (from `schemas/reports.py`)
  - On `ValueError("no_items")`: return HTTP 422 with `"No published auction items found for the selected criteria"`
  - Returns `StreamingResponse` with `application/pdf` and `Content-Disposition: attachment; filename="bid-cards-{size}-{date}.pdf"`

### Frontend for User Story 2

- [ ] T016 [P] Create `frontend/fundrbolt-admin/src/components/reports/BidCardSizeDialog.tsx`:
  - Radix `Dialog` component with size picker
  - 4 options displayed as cards/radio buttons: 2"×3", 2"×4", 3"×3", 3"×5" with visual size preview (scaled rectangles)
  - "Generate & Download" button triggers download; "Cancel" dismisses
  - Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `onConfirm: (size: LabelSize) => void`, `isLoading: boolean`
  - Default selected: 3"×5"

- [ ] T017 Extend `frontend/fundrbolt-admin/src/features/events/auction-items/AuctionItemsIndexPage.tsx`:
  - Add per-row checkbox (`selectedItemIds: Set<string>`, managed with `useState`)
  - Add "Select All" checkbox in table header (selects all visible published items)
  - Add toolbar: **R004**: button always labeled "Print Bid Cards" (or "Print N Bid Cards" when N items selected); when 0 checked, passes `item_ids: null` to print all items
  - Clicking "Print Bid Cards" opens `BidCardSizeDialog`
  - On dialog confirm: call `reportService.downloadBidCards(currentEvent.id, { item_ids: [...selectedItemIds], label_size })`; if no items checked, pass `item_ids: null` (all items)
  - Wrap with `DownloadReportButton`-style loading overlay (or reuse component with headless pattern)
  - Empty state: if 0 published items, show message "No published auction items. Publish items before printing bid cards." and disable "Print Bid Cards"

**Checkpoint**: Navigate to auction items → select 2 items → Print Bid Cards → choose 3"×5" → PDF downloads with 2 pages at exactly 76.2mm × 127mm

## Phase 5: User Story 3 — Auctioneer Financial Report (Priority: P3)

**Goal**: Auctioneer downloads a commission/earnings PDF from their dashboard.

**Independent Test**: `GET /api/v1/admin/events/{event_id}/auctioneer/report` (as auctioneer role) returns a valid PDF with commission totals.

### Backend for User Story 3

- [ ] T018 Create `backend/app/services/auctioneer_report_service.py`:
  - `AuctioneerReportService.__init__(db: AsyncSession)`
  - `async generate_pdf(event_id: UUID, auctioneer_user_id: UUID, auctioneer_display_name: str) -> bytes`
  - Calls `AuctioneerService(db).get_dashboard(event_id, auctioneer_user_id)` for `DashboardResponse`
  - Calls `AuctioneerService(db).get_commissions(auctioneer_user_id, event_id)` for `CommissionListResponse`
  - Calls `AuctioneerService(db).get_event_settings(auctioneer_user_id, event_id)` for `AuctioneerEventSettingsResponse`
  - Loads event name/date via direct ORM query (Event model)
  - Renders `templates/reports/auctioneer_report.html`
  - WeasyPrint in executor → PDF bytes
  - When `EarningsSummary.total_earnings == 0` and no commissions configured: includes a visible note in template context
- [ ] T019 Create `backend/app/templates/reports/auctioneer_report.html`:
  - Letter/A4 page (`@page { size: letter; margin: 15mm; }`)
  - Header: event name, event date, "Auctioneer: {name}", "Report Generated: {timestamp}"
  - Section 1 — Event Financial Summary: table with Live Auction Total, Silent Auction Total, Paddle Raise Total, Revenue Generators Total (if present), Event Grand Total (bold)

  - Section 4 — Grand Total Box: prominently styled box showing total auctioneer commission (sum of category + per-item)
