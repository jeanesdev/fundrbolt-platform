# Data Model: PWA Capabilities

**Feature**: 032-pwa-capabilities | **Date**: 2026-03-05 | **Phase**: 1

## Overview

This feature is **entirely frontend**. There are no new database tables, API endpoints, or backend models. The "data model" for PWA capabilities consists of client-side caches managed by Workbox and browser-local state in `localStorage`.

## Entities

### 1. Precache Manifest (Workbox-managed)

The set of app shell assets cached during service worker installation. Automatically generated by `vite-plugin-pwa` from the build output.

| Field | Type | Description |
|-------|------|-------------|
| `url` | string | Relative URL of the cached asset |
| `revision` | string \| null | Content hash for cache-busting (null for hashed filenames) |

**Storage**: CacheStorage API, cache name `workbox-precache-v2-{origin}`
**Lifecycle**: Populated on SW install; updated when new SW activates
**Contains**: HTML entry point, JS/CSS bundles, key static assets (logo, icons, offline.html)

### 2. Runtime Image Cache

Dynamically cached image responses using a CacheFirst strategy.

| Property | Value |
|----------|-------|
| Cache name | `images-cache` |
| Strategy | CacheFirst |
| Max entries | 200 |
| Max age | 30 days |
| Size cap | ~50 MB (enforced via entry count) |
| Eviction | LRU (least recently used) |
| Purge on quota error | Yes |
| URL pattern | `\.(?:png\|jpg\|jpeg\|gif\|svg\|webp)$` |

**Contains**: Auction item photos, sponsor logos, event banners, user-uploaded images

### 3. Runtime API Cache

Dynamically cached API responses using a NetworkFirst strategy.

| Property | Value |
|----------|-------|
| Cache name | `api-cache` |
| Strategy | NetworkFirst |
| Network timeout | 5 seconds |
| Max entries | 100 |
| Max age | 24 hours |
| Size cap | ~10 MB (enforced via entry count) |
| Eviction | LRU |
| Purge on quota error | Yes |
| URL pattern | `/api/v1/.*` |

**Contains**: Event details, auction items, bid data, registration info, sponsor lists

### 4. Google Fonts Cache

Font files cached with StaleWhileRevalidate strategy.

| Property | Value |
|----------|-------|
| Cache name | `google-fonts-cache` |
| Strategy | StaleWhileRevalidate |
| Max entries | 30 |
| Max age | 1 year |
| URL pattern | `fonts\.googleapis\.com\|fonts\.gstatic\.com` |

**Contains**: Inter and Manrope font files (WOFF2)

### 5. Install Prompt State (localStorage)

Browser-local state tracking install prompt dismissal.

| Key | Type | Description |
|-----|------|-------------|
| `pwa-install-dismissed-donor` | `{ dismissedAt: number }` | Timestamp of last dismissal (Donor PWA) |
| `pwa-install-dismissed-admin` | `{ dismissedAt: number }` | Timestamp of last dismissal (Admin PWA) |

**Cooldown**: 7 days (604,800,000 ms)
**Cleared**: Never (naturally expires via cooldown check)

### 6. Service Worker Update State (localStorage)

Tracks when a pending update was first detected for the 24-hour auto-update timer.

| Key | Type | Description |
|-----|------|-------------|
| `sw-update-detected` | `string` (timestamp) | When the pending update was first noticed |

**Cleared**: After update is applied

## Entity Relationships

```
Service Worker
├── manages → Precache Manifest (app shell assets)
├── manages → Runtime Image Cache (CacheFirst)
├── manages → Runtime API Cache (NetworkFirst)
├── manages → Google Fonts Cache (StaleWhileRevalidate)
└── serves → Offline Fallback Page (from precache on navigation failure)

Browser localStorage
├── Install Prompt State (7-day dismissal cooldown)
└── SW Update State (24-hour auto-update timer)
```

## No Database Changes

- No new tables
- No new columns
- No Alembic migrations
- No backend API changes
- All data is client-side, ephemeral, and managed by the browser/Workbox
